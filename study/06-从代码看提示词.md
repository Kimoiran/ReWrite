# 第六课：从代码看提示词（实战阅读）

这一课我们打开实际的提示词文件，看它长什么样、怎么工作、怎么修改。

## 6.1 提示词文件长什么样

**文件：`prompt_templates.py`**

```python
"""提示词模板 + 架构说明书。"""

SKILL_MAP = """
工具列表（共 29 个）：

📖 章节
  - get_chapters / read_chapter / create_chapter / ...
👤 人物卡
  - get_character_groups / get_characters / ...
📋 大纲
  - get_outline / update_outline_entry / delete_outline_entry
📅 时间线
  - create_timeline_event / get_timeline / ...
🌍 世界观
  - get_worldview / create_worldview_entry / ...
🗺️ 地图
  - get_map / create_map_node / update_map_node / ...

work 参数由系统自动注入，不需要填写。

响应规则（每次回复前必须检查）：
1. 用户提到作品中任何内容时 → 先调对应的 get 工具再回答
2. 用户说要"写""改""创建""删除" → 调对应的工具执行
3. 禁止说"我先做A再做B"——直接用工具做
4. 禁止说"以下是完整内容"——直接用工具写入
5. 禁止「我来读取/检查/操作」——直接调工具
6. 每次只输出工具调用，等结果再决定下一步
7. 未找到且用户要创建 → 调 create
8. 函数名必须完全精确
"""

DEFAULT_SYSTEM_PROMPT = """你是一个写作助手。你的工作方式不是打字，是调用工具。

""" + SKILL_MAP
```

这个文件定义了 **AI 的「人格」**——它决定了 AI 怎么理解自己的角色、有什么能力、按什么规则行动。

## 6.2 提示词是怎么传给 AI 的

### 入口：用户发消息

当你在聊天框输入消息并点击发送时，调用链如下：

```
chat_panel.py 用户点击发送
    │
    ▼
module.py _on_chat_message(message, scope)
    │  1. 获取上下文数据
    │  2. 组装提示词
    ▼
agent.py send_message(message, current_context)
    │  1. system = DEFAULT_SYSTEM_PROMPT + context
    │  2. 发给 API
    ▼
providers.py API 调用
    {model, messages: [{role: "system", content: system}, {role: "user", content: message}], tools: [...]}
```

### 具体代码

在 `module.py` 中：

```python
def _on_chat_message(self, message: str, scope: str):
    # 1. 获取上下文（人物卡/大纲/时间线等的概览数据）
    ctx = self.get_context(scope)
    
    # 2. 发给 worker 处理
    self._proposal_worker = ToolProposalWorker(self.agent, message, ctx)
    self._proposal_worker.start()
```

在 `worker.py` 中：

```python
def run(self):
    from .providers import get_proposals_only
    result = get_proposals_only(self.agent, self.message, self.context)
    ...
```

在 `providers.py` 中：

```python
def get_proposals_only(agent, message, context):
    # 1. 从配置或默认值获取提示词
    system = config.get("system_prompt", "") or DEFAULT_SYSTEM_PROMPT
    
    # 2. 把上下文拼接到提示词后面
    if context:
        system += f"\n\n## 当前作品上下文\n{context}"
    
    # 3. 获取所有 Skill 的工具定义
    tools = get_openai_tools()
    
    # 4. 组装请求体
    full = [{"role": "system", "content": system}]
    full.extend(recent_messages)
    
    body = {
        "model": "deepseek-chat",
        "messages": full,
        "tools": tools,        # ← Skill 的结构化定义
    }
    
    # 5. 调 API
    response = requests.post(api_url, json=body)
    return response
```

## 6.3 上下文的拼接

上下文数据不是写死在提示词里的，是**实时从各模块读取**的。

在 `contexts.py` 中：

```python
def collect_context(scope, chapter_module, character_module, ...):
    parts = []
    
    if "characters" in scope and character_module:
        texts = _characters_text(character_module.nodes)
        parts.append(f"<<<人物设定卡概览>>>\n{texts}")
    
    if "outline" in scope and outline_module:
        texts = _outline_text(outline_module.entries)
        parts.append(f"<<<大纲>>>\n{texts}")
    
    if "map" in scope and map_module:
        texts = _map_text(map_module.nodes, map_module.routes)
        parts.append(f"<<<🗺️ 地图>>>\n{texts}")
    
    return "\n\n".join(parts)
```

最终拼接后的提示词长这样：

```
你是一个写作助手。你的工作方式不是打字，是调用工具。

工具列表（共 29 个）：
...

响应规则：
...

## 当前作品上下文
<<<人物设定卡概览>>>
📁 自然之力
  👤 主角（十八岁·自然之力代行者）

<<<🗺️ 地图>>>
📍 北方王国 (country)
    📍 边陲小镇 (city)

<<<大纲>>>
▶ 第一章 [待写]
  ...
```

整个过程在代码中表现为：

```
DEFAULT_SYSTEM_PROMPT      ← 提示词模板
    + "\n\n## 当前作品上下文\n"
    + collect_context(...)  ← 实时数据
    = 最终发给 AI 的系统提示词
```

## 6.4 提示词的控制面板

用户可以通过 AI 面板上的**芯片开关**来控制 AI 能读到什么上下文：

```
┌──────────────────────────────────┐
│ AI 可以读取：                      │
│ [当前章节] [选中文本] [大纲]       │
│ [人物设定卡] [时间线] [世界观]     │
│ [🗺️ 地图] [作品信息]              │
│                                   │
│ [写作助手] [深度分析] [灵感发散]   │ ← 预设
└──────────────────────────────────┘
```

每个芯片对应 `scope` 中的一个模块。芯片的开关状态决定了 `collect_context()` 会收集哪些数据。

## 6.5 如何修改提示词

如果你想修改 AI 的行为，改 `prompt_templates.py` 中的 `SKILL_MAP` 即可。

### 常见修改场景：

| 你想做什么 | 修改什么 |
|-----------|---------|
| 让 AI 更积极地调工具 | 加一条规则或强化已有规则 |
| 添加新模块的 Skill | 在工具列表加一行 |
| 改变 AI 的回复风格 | 修改 `DEFAULT_SYSTEM_PROMPT` 的第一句话 |
| 让 AI 少犯错 | 加具体的规则和示例 |

**注意：** 每次修改后需要重新打包（或重启）才能生效。

## 6.6 调试技巧

### 如何看 AI 实际收到了什么？

可以在 `providers.py` 中加一行打印：

```python
def get_proposals_only(agent, message, context):
    system = config.get("system_prompt", "") or DEFAULT_SYSTEM_PROMPT
    if context:
        system += f"\n\n## 当前作品上下文\n{context}"
    
    print("=== SYSTEM PROMPT ===")  # ← 加这行
    print(system[:500])             # ← 打印前 500 字
    print("=== END ===")
    
    full = [{"role": "system", "content": system}]
    ...
```

这样你在控制台就能看到每次发给 AI 的完整提示词。

## 6.7 小结

- 提示词定义在 `prompt_templates.py` 中的 `DEFAULT_SYSTEM_PROMPT`
- 上下文数据实时从各模块读取，通过 `collect_context()` 拼接
- 用户通过芯片开关控制 AI 能读到什么
- 修改提示词 = 修改 AI 的行为
- 提示词 + Skill 定义（tools 参数）+ 上下文一起发给 API
