# 第一课：什么是 AI Agent？

## 1.1 先做一个思想实验

想象你有一个**特别聪明的助手**，他读过万卷书，上知天文下知地理。但是——他有一个致命的缺陷：

**他没有手。**

你让他「帮我把桌上那本书拿过来」，他只能回答你：「好的，那本书在桌上，请你走过去，伸出右手，用拇指和食指捏住书脊，轻轻拿起来。」

他**知道**怎么做，但他**做不到**。因为他没有手。

这就是普通 AI（大语言模型）的现状——它知道很多，但无法操作任何东西。

## 1.2 给 AI 装上手

**AI Agent（人工智能体）** 就是给 AI 装了「手」。这个"手"就是我们写的代码函数。

```
普通 AI：用户 → AI（只能说话） → 文字回复

AI Agent：用户 → AI（思考决定） → 调用函数（执行操作） → 返回结果 → 文字回复
```

### 生活中的例子

| 场景                           | 普通 AI            | AI Agent                 |
| ------------------------------ | ------------------ | ------------------------ |
| 「帮我订一张明天去北京的机票」 | 给你一段订票攻略   | 直接打开订票网站帮你操作 |
| 「把主角的年龄改成二十岁」     | 告诉你应该怎么改   | 直接调函数修改数据文件   |
| 「检查一下第一章的字数」       | 告诉你字数的重要性 | 直接读取文件给你报告     |

在 ReWrite 中，我们给 AI 装上了 30 只"手"（30 个 Skill）。每个 Skill 都能执行一个具体的操作。

## 1.3 AI Agent 的三个核心组件

一个 AI Agent 由三部分组成，缺一不可：

```
┌─────────────────────────────────────────────────────┐
│                     AI Agent                         │
│                                                     │
│  ┌─────────────────────┐                            │
│  │     大脑（LLM）      │  ← 理解语言、做决策          │
│  │  DeepSeek / Claude  │     知道「改年龄」要调什么    │
│  └──────────┬──────────┘                            │
│             │ 决定调用哪个函数                        │
│             ▼                                       │
│  ┌─────────────────────┐                            │
│  │     手脚（Skills）   │  ← 30 个可执行函数           │
│  │  get_characters     │     每个函数做一件具体的事    │
│  │  update_character   │                            │
│  │  search_chapters    │      ← 新增的 RAG 搜索       │
│  └──────────┬──────────┘                            │
│             │ 执行结果返回                            │
│             ▼                                       │
│  ┌─────────────────────┐                            │
│  │     记忆（Memory）   │  ← 对话历史 + 作品数据       │
│  │  关了再开还记得      │     持久化到磁盘文件         │
│  └─────────────────────┘                            │
└─────────────────────────────────────────────────────┘
```

### 1.3.1 大脑（LLM）

LLM 是 Large Language Model（大语言模型）的缩写。你可以把它理解为一个**极度聪明但没有任何常识的同事**：

- ✅ 它懂语法、逻辑、推理
- ✅ 它能理解复杂的自然语言
- ✅ 它能从上下文中学习
- ❌ 它不知道自己能做什么（需要提示词告诉它）
- ❌ 它不能直接操作文件/数据库（需要 Skill 帮它）

在 ReWrite 中，LLM 由 `agent.py` 中的 `AIAgent` 类管理：

```python
class AIAgent:
    def __init__(self, config, work_name):
        self.history = []  # 对话记忆
        # 加载历史
        saved = load_chat_history(work_name)
        if saved:
            self.history = saved
```

### 1.3.2 手脚（Skill）

Skill 是 AI 的"手"。每个 Skill 是一个 Python 类，继承自 `Skill` 基类：

```python
class UpdateCharacterSkill(Skill):
    name = "update_character"  # 函数名
    description = "修改角色字段值"  # 告诉 AI 这个函数的作用
  
    def execute(self, args):
        # 1. 读取 JSON 文件
        # 2. 找到角色
        # 3. 修改字段值
        # 4. 保存文件
        return {"success": True}
```

**AI 自己不会执行代码**——它只负责决定「应该调哪个函数」。实际的执行由 Python 代码完成。

### 1.3.3 记忆（Memory）

记忆分为两部分：

1. **对话历史**：用户和 AI 的聊天记录，保存在 `~/.rewrite/ai_history/` 目录下
2. **作品数据**：人物卡 JSON、章节 HTML、大纲 JSON 等

对话历史的持久化流程：

```
用户发消息 → AI 回复 → 追加到 self.history → 写入磁盘文件
                ↓
下次打开软件 → 读取磁盘文件 → 恢复到 self.history → AI 记得之前聊过什么
```

代码层面看：

```python
# 保存记忆
def _persist(self):
    save_chat_history(self.work_name, self.history)
    # 写入 ~/.rewrite/ai_history/{作品名}.json

# 恢复记忆
def __init__(self, config, work_name):
    saved = load_chat_history(work_name)
    if saved:
        self.history = saved  # 恢复上次的对话
```

## 1.4 AI Agent 的工作流程（完整版）

当你在 AI 面板输入「把主角的年龄改成二十岁」时，背后发生的事情：

```
第 1 步：你输入消息
  
    chat_panel.py 捕获输入
        ↓
第 2 步：组装请求
  
    module.py 收集上下文（人物卡概览、章节概览等）
    + prompt_templates.py 中的提示词
    + 30 个 Skill 的定义
    → 打包成 API 请求
        ↓
第 3 步：发给 AI
  
    通过 HTTP 请求发送到 DeepSeek/Claude 的 API
    请求体包含：system prompt + 用户消息 + 工具定义
        ↓
第 4 步：AI 思考
  
    大语言模型内部推理：
    「用户要修改主角的年龄」
    「我需要先看看主角是否存在」
    「我可以调用 get_characters 工具」
        ↓
第 5 步：AI 返回工具调用（不是文字）
  
    {
      "tool_calls": [{
        "function": {"name": "get_characters", "arguments": '{"group": "自然之力"}'}
      }]
    }
        ↓
第 6 步：系统执行 Skill
  
    registry.py 找到 GetCharactersSkill
    执行 → 读取 characters.json → 返回结果
        ↓
第 7 步：结果送回 AI
  
    AI 看到主角确实存在
    决定下一步：调 update_character
        ↓
第 8 步：弹出确认框
  
    聊天框显示「AI 请求操作：将主角的年龄修改为二十岁」
    你点击「允许」
        ↓
第 9 步：执行修改
  
    UpdateCharacterSkill.execute() 执行
    改 JSON 文件 → 保存 → 返回结果
        ↓
第 10 步：AI 回复
  
    AI 生成文字：「✅ 已将「主角」的「age」修改为「二十岁」」
    你看到这条消息
```

这个流程中，**第 4-7 步会循环多次**，直到 AI 认为任务完成。这就是**多轮递归**。

## 1.5 多轮递归的循环过程

因为 AI 一次只能调用一个工具（或几个工具），复杂的任务需要多轮对话：

```
用户：「帮我看看主角和一位长者的背景有没有矛盾」

    第 1 轮：AI 调 get_characters(group="自然之力") → 看到主角
    第 2 轮：AI 调 get_characters(group="知识") → 看到一位长者
    第 3 轮：AI 分析两者背景 → 发现矛盾
    第 4 轮：AI 回复「主角说...但一位长者说...这里有矛盾」
```

每一轮 AI 都：

1. 看到上一轮工具返回的结果
2. 决定下一步做什么
3. 要么继续调工具，要么生成最终回复

这就是**多轮递归**的核心——AI 不断「思考→行动→观察结果→再思考」直到任务完成。

## 1.6 Agent 的边界

AI Agent 不是万能的。在 ReWrite 中，Agent 能做和不能做的事：

| 能做的事                           | 不能做的事                       |
| ---------------------------------- | -------------------------------- |
| 读取/修改作品中所有模块的数据      | 访问互联网（它只能访问本地文件） |
| 调用 30 个预定义的 Skill           | 执行不在 Skill 列表中的操作      |
| 记住对话历史（关了再开还在）       | 记住其他作品的数据               |
| 自动修正错误（找不到时换一种方式） | 在没有 Skill 的情况下创造新功能  |

## 1.7 小结

- **AI Agent** = LLM（大脑）+ Skill（手脚）+ Memory（记忆）
- Agent 的工作方式是：**理解 → 决策 → 执行 → 观察 → 再决策 → 完成**
- **多轮递归**让 AI 能执行复杂的多步骤任务
- Agent 的边界由**可用的 Skill 列表**决定——Skill 越多，Agent 越强大
