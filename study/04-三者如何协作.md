# 第四课：三者如何协作？

## 4.1 完整对话流程

现在我们用前三课的知识，追踪一次完整的 AI 对话在 ReWrite 内部是如何流动的。

### 场景：你说「帮我把主角的年龄改成二十岁」

```
第 1 步：你输入消息
─────────────────────
你在 AI 聊天面板中打字：「帮我把主角的年龄改成二十岁」
点击「发送」按钮。

负责文件：chat_panel.py
```

```
第 2 步：组装请求
─────────────────────
module.py 收到消息后：
  1. 从 contexts.py 获取当前作品的数据概览
  2. 从 prompt_templates.py 获取系统提示词
  3. 从 registry.py 获取 30 个工具的定义
  4. 打包成 API 请求

最终发给 API 的数据（简化）：
──────────────────────────────────
messages = [
  {"role": "system", "content": "你是一个写作助手..."
   + "\n\n## 当前作品上下文\n<<<人物卡概览>>>\n📁 自然之力\n  👤 主角..."},
  {"role": "user", "content": "帮我把主角的年龄改成二十岁"}
]
tools = [{name:"get_characters", description:"获取人物卡", ...},
         {name:"update_character", description:"修改角色", ...},
         ...]  // 30 个工具

负责文件：module.py -> contexts.py -> prompt_templates.py -> registry.py
```

```
第 3 步：AI 收到请求
─────────────────────
API 服务器（DeepSeek/Claude）收到请求。
AI 模型阅读 system prompt：
  → 我是写作助手
  → 我有 30 个工具
  → 规则：用户说"改"就调 update
  → 上下文：人物卡概览里有主角

AI 理解用户消息：
  → 用户要修改主角的年龄
  → 规则要求我直接调工具
  → 但我需要先确认主角是否存在

AI 做出决策：
  → 第一步：调 get_characters 看看主角在不在
  → 第二步：如果存在就调 update_character

AI 返回工具调用请求：
──────────────────────────────────
{
  "choices": [{
    "message": {
      "content": null,
      "tool_calls": [{
        "id": "call_xxx",
        "function": {
          "name": "get_characters",
          "arguments": "{\"group\": \"自然之力\"}"
        }
      }]
    }
  }]
}
```

```
第 4 步：系统执行工具
─────────────────────
收到 API 返回后，module.py 判断：
  → 不是文字回复
  → 是工具调用请求（get_characters）
  → 不弹出确认框（get 操作不需要确认）
  → 直接执行

查找并执行：
  registry.py -> get_skill("get_characters") -> GetCharactersSkill
  执行 -> 读取 characters.json -> 找到自然之力分组下的角色
  返回 -> {nodes: [{name:"主角", age:"十八岁", ...}, ...]}

把结果送回 AI：
──────────────────────────────────
messages 追加：
  {"role": "assistant", "tool_calls": [...]}  // AI 刚才的请求
  {"role": "tool", "content": "主角: 年龄=十八岁, 性格=..."}  // 工具结果
```

```
第 5 步：AI 继续决策
─────────────────────
AI 看到主角存在，年龄是十八岁。
用户要求改成二十岁。
AI 决定调 update_character。

AI 再次返回工具调用请求：
──────────────────────────────────
{
  "tool_calls": [{
    "function": {
      "name": "update_character",
      "arguments": "{\"name\": \"主角\", \"field\": \"age\", \"value\": \"二十岁\"}"
    }
  }]
}
```

```
第 6 步：弹出确认框
─────────────────────
module.py 判断：
  → 这是 update 操作（修改数据）
  → 需要用户确认
  → 弹出确认气泡

你在聊天框看到：
──────────────────────────────────
┌──────────────────────────────┐
│ AI 请求操作                   │
│                              │
│ • 已将「主角」的「age」       │
│   修改为「二十岁」            │
│                              │
│     [取消]    [允许]          │
└──────────────────────────────┘
```

```
第 7 步：你点击「允许」
─────────────────────
module.py 收到确认信号：
  → 执行 UpdateCharacterSkill
  → 读取 characters.json
  → 找到主角
  → 把 age 改成「二十岁」
  → 保存文件
  → 返回结果给 AI

结果：
──────────────────────────────────
{
  "success": True,
  "old": "十八岁",
  "new": "二十岁"
}

同时：
  → 刷新人物卡面板（_refresh_panels）
  → 重建 RAG 索引（如果章节有变化）
```

```
第 8 步：AI 生成最终回复
─────────────────────
AI 看到工具执行成功：
  → ✅ 修改已生效
  → 生成回复文字

你看到：
──────────────────────────────────
✅ 已将「主角」的「age」修改为「二十岁」

同时，人物卡面板中的「主角」年龄也自动更新了。
```

## 4.3 代码调用链（从输入到输出）

```
你点击发送
  │
  ▼
chat_panel.py:_on_send()
  │ 发送 send_message_signal
  ▼
module.py:_on_chat_message()
  │ 1. get_context(scope)     ← 收集上下文
  │ 2. ToolProposalWorker()   ← 创建后台线程
  ▼
worker.py:run()
  │ 后台线程运行
  ▼
providers.py:get_proposals_only()
  │ 1. DEFAULT_SYSTEM_PROMPT + context → system prompt
  │ 2. get_openai_tools() → 30 个工具定义
  │ 3. 调用 API
  ▼
API 返回
  │
  ├─ 工具调用 → module.py:_on_tool_proposals()
  │              │
  │              ├─ get → 直接执行
  │              └─ update/delete → 显示确认气泡
  │                                │
  │                   用户点击「允许」
  │                                │
  │                    module.py:_execute_and_continue()
  │                    ├─ registry.py:execute_skill()
  │                    ├─ _refresh_panels()
  │                    └─ _tool_loop() → 继续 AI 循环
  │
  └─ 文字回复 → module.py:_on_ai_response()
                ├─ 显示在聊天框
                └─ 创建批注（如果用户要求分析）
```

## 4.4 三者的关系总结

```
┌────────────────────────────────────────────────────────────┐
│                     提示词（Prompt）                        │
│                                                           │
│  告诉 AI：                                                 │
│  - 你是谁（写作助手）                                       │
│  - 你有什么（30 个工具）                                    │
│  - 怎么工作（规则 1-8）                                     │
│                                                           │
│  存在 prompt_templates.py 中                                │
└────────────────────┬──────────────────────────────────────┘
                     │ AI 读取
                     ▼
┌────────────────────────────────────────────────────────────┐
│                    AI Agent                                 │
│                                                           │
│  1. 理解提示词 → 知道自己是「用工具工作的写作助手」           │
│  2. 理解上下文 → 看到人物卡、地图等数据概览                  │
│  3. 理解用户消息 → 「把主角的年龄改成二十岁」               │
│  4. 决策 → 调 update_character 工具                        │
│  5. 输出工具调用 → {name: "update_character", args: {...}} │
│                                                           │
│  存在 agent.py + providers.py 中                            │
└────────────────────┬──────────────────────────────────────┘
                     │ 系统执行
                     ▼
┌────────────────────────────────────────────────────────────┐
│                    Skill 系统                                │
│                                                           │
│  1. registry.py 找到对应 Skill                              │
│  2. Skill.execute(args) 执行                               │
│     ├─ 读/写 JSON 文件                                      │
│     ├─ 返回 {success, data}                                 │
│  3. summarize() → 转成文字描述                               │
│  4. 结果返回给 AI Agent                                    │
│                                                           │
│  存在 skills/ 目录下，30 个文件                               │
└────────────────────────────────────────────────────────────┘
```

## 4.5 角色类比

| 角色 | 类比 | 在项目中的代码 |
|------|------|--------------|
| **提示词** | 员工的入职手册 | `prompt_templates.py` |
| **AI Agent** | 员工本人 | `agent.py` + `worker.py` + `providers.py` |
| **Skill** | 员工手里的工具 | `skills/*.py` |
| **注册表** | 工具柜上的标签 | `skills/registry.py` |
| **RAG 引擎** | 图书管理员（帮忙找资料） | `rag.py` |
| **上下文** | 贴在墙上的当前项目状态 | `contexts.py` |
| **确认框** | 操作前的审批流程 | `module.py` |
| **模块入口** | 项目经理（协调所有环节） | `module.py` |

## 4.6 小结

- 一次完整的 AI 对话涉及 **8 个步骤**
- 用户输入 → 组装请求 → AI 决策 → 调工具 → 确认 → 执行 → 刷新 → 回复
- 中间 **第 3-5 步循环**，直到 AI 认为任务完成（多轮递归）
- **提示词**驱动 AI 的行为
- **AI Agent**负责决策和循环
- **Skill 系统**负责具体执行
- **RAG 引擎**辅助搜索正文内容
- 所有操作**用户确认后才执行**，安全可控
